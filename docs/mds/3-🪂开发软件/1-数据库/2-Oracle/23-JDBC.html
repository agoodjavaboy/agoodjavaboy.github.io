JDBC ​

  • Java Data Base Connectivity,java数据库连接。
  • JDBC是java连接数据库的技术，可以利用java程序在可以为多种关系数据库提供统一访问中存取值，进行数据操作。
  • JDBC需要第三方jar包的支持，通常不同的数据库厂商都会发行很多版本的JDBC的jar包。

六步骤 ​

  1. 注册驱动

    Class类的forName方法根据指定的全类名进行类加载，类加载时会执行其中的静态代码块，从而进行驱动注册。

    java
    Class.forName("OracleDriver全类名");
//OracleDriver类存在于导入的jar包中：oracle.jdbc.driver.OracleDriver

  2. 获得连接

    • jdbc：协议名
    • oracle：子协议名
    • thin：驱动模式，还有oci模式（必须在客户机上安装客户端）
    • 127.0.0.1：默认的本地IP
    • 1521：数据库默认的端口
    • 数据库名称：10G可用XE，11G可用orcl
    java
    Connection conn = DriverManager.getConnection("url","username","password");
/*
url：jdbc:oracle:thin:@127.0.0.1:1521:数据库名称
username：链接数据库用户名
password：用户名所对应的密码
*/

  3. 获得编译对象

    java
    Statement sta = conn.createStatement();

  4. 执行SQL指令

    java
    //查看是否有结果集
boolean result = sta.excute("sql语句");

//获取结果集，保存在rs对象中
ResultSet rs = sta.excuteQuery("sql语句");

//获得修改/增加/删除的条目总数，保存在rows变量中
int rows = sta.excuteUpdate("sql语句");

  5. 处理结果集

    • next：移动指针到下一条记录并查看此记录是否有效。
    • getXXX：get系列方法用于提取单独一列中的某一单元格中的值。
    java
    //当查询操作时，结果集将保存在在ResultSet对象中
while(rs.next()){//next方法类似迭代器的指针移动方法
    int id = rs.getInt("列名")；
    String name = rs.getString("列名");
}

  6. 释放资源

    java
    //先释放最后开启的资源，再递归释放
rs.close();
sta.close();
conn.close();

SQL注入 ​

  • 通过特殊的SQL语句对原有的SQL进行篡改。
  • 一般情况下写：or 1=1，表示绝对执行。
  • 使用JDBC中的Statement编译对象就容易遭受SQL注入攻击。
java
// 以下为用户登录操作
	public Student getUserByNameAndID(Student s){
		Connection conn = null;
		Statement sta = null;
		ResultSet rs = null;

		Student resultUser = null;
		try {
			Class.forName("oracle.jdbc.driver.OracleDriver");
			conn = DriverManager.getConnection("jdbc:oracle:thin:@localhost:1521:orcl","hr","system");
			sta = conn.createStatement();//使用Statement编译对象进行SQL编译
			String sql = "select * from student2 where name='"+s.getName()+"' and id="+s.getId();
			System.out.println(sql);
			rs = sta.executeQuery(sql);
			if(rs.next())
				resultUser = new Student(rs.getInt(1),rs.getString("NAME"),rs.getInt(3),rs.getString("sex"));
		} catch (ClassNotFoundException e) {
			e.printStackTrace();
		} catch (SQLException e) {
			e.printStackTrace();
		} finally{
			try {
				if(rs!=null)rs.close();
				if(sta!=null)sta.close();
				if(conn!=null)conn.close();
			} catch (SQLException e) {
				e.printStackTrace();
			}
		}
		return resultUser;
	}

预编译 ​

  • PreparedStatement是Statement的子类。
  • 其两者都是通过Connection创建的。
  • PreparedStatement在创建时，需要传入SQL语句，但在执行SQL语句时不用传入SQL语句，Statement与其相反。
  • PreparedStatement执行同构SQL效率高，Statement执行异构SQL效率高。
  • Oracle接收到java发送指令后执行顺序：
    1. 检测语法。
    2. 检测权限：用户对表的操作权限。
    3. 将SQL转换为内部指令。
    4. 执行指令。
  • PreparedStatement若执行五条SQL，在创建的时候先做前几步进行预编译，在最后执行的时候执行最后几步。Statement每执行一条SQL都将执行这几步。
  • PreparedStatement可以将SQL语句中的值以占位符的形式存在，使SQL语句更加灵活安全。
  • PreparedStatement可以有效防止SQL注入问题，Statement将传入的内容作为SQL的一部分，而PreparedStatement是将传入的内容作为一个值，不会影响整体的SQL语句。
java
//获得预编译对象
conn.prepareStatement("存在占位符的SQL语句");
//select * from users where name=? and password=?

//替换占位符：前面的数字表示第几个占位符
ps.setString(1,name);
ps.setString(2, psw);
ps.setDouble(3, balance);

批处理 ​

  • 可以将多条同构的SQL语句加载到内存中一次性执行。

  • 批处理可以减少与数据库交互的次数，从而提高执行效率。

  • Statement的批处理在addBach时要添加进SQL语句，PreparedStatement不用，直接调用addBach方法即可。

  • Statement对象批处理返回值表示每条记录执行后所影响的条目数。PreparedStatement对象批处理的结果为处理结果：

    -2：成功执行未出现异常，但处理条目数未知。

    -3：执行失败出现异常。

  • 批处理一般用于增删改，无法用于查询。

java
//注册驱动
Class.forName("驱动全类名");

//获得连接
Connection conn = DriverManager.getConnection("url","username","password");

//SQL语句预编译
PreparedStatementps = conn.perpareStatement("存在占位符的sql语句");

//set方式替换占位符第一次
ps.setString(1,"值");
ps.setInt(2,值);

ps.addBach();//批处理加载第一次
//set方式替换占位符第二次
ps.setString(1,"值");
ps.setInt(2,值);

ps.addBach();//批处理加载第二次

//执行批处理
int[] results = ps.executeBatch();
//结果集为：多条SQL语句执行结果：-2或者-3


//Statement执行方式：ls表示一个包含多个Student对象的集合
s = conn.createStatement();
for(Student s:ls){
    s.addBatch("delete from student2 where id="+s.getId());
}

int[] resultArr = s.executeBatch();
//结果集为：多条SQL语句影响条目数：0或者1

事务控制 ​

  • JDBC默认事务为自动事务，及执行后只要没有错误自动提交。
  • 但在多条SQL存在原子性问题时，需要手动的控制事务。
  • 书写位置：
    • 提交操作一般写在try块的最底部。
    • 回滚操作一般写在catch块的第一步。
    • 事务控制代码一般会与六步骤分离。
java
//关闭自动提交方式
conn.setAutoCommit(false)
//任意多次地调用ExecuteUpdate方法
stmt.executeUpdate(command1);
stmt.executeUpdate(command2);
stmt.executeUpdate(command3);
//当所有命令都已经执行后，调用commit方法
conn.commit();
//如果出现一个错误，则调用下面的方法
conn.rollback();
java
//模拟转账流程，增加事务管理功能

//转账之：业务逻辑模块：service.StudentService.
/*
	+ 因为事务管理依赖Connection，所以需要将Connection放到service层
	+ 要保证事务中的多条SQL语句都是通过一个Connection运行的
	+ 如果还在DAO层创建Connection，那绝对无法保证使用的是同一个连接对象
	+ 创建连接之前需要“注册驱动”，所以将驱动注册流程也写到了service层
*/
public void zhuanZhang(int from,int to,int age){
    Connection conn = null;
    try {
        Class.forName("oracle.jdbc.driver.OracleDriver");
        conn = DriverManager.getConnection("jdbc:oracle:thin:@127.0.0.1:1521:orcl","hr","system");
        conn.setAutoCommit(false);		//将自动事务关闭

        //通过参数传递，保证两个方法中所使用的连接对象是同一个
        sd.updateAgeByIDAndNum_jian(from, age,conn);		//调用减寿命的方法
        sd.updateAgeByIDAndNum_jia(to, age,conn);		//调用增加寿命的方法

        conn.commit();//如果以上语句中没有任何一条出现异常，则执行提交操作
    } catch (Exception e) {
        try {
            conn.rollback();//如果try块中的语句，任何一个出现异常都将回滚两条SQL的运行
        } catch (SQLException e1) {
            e1.printStackTrace();
        }
        e.printStackTrace();
    }
}

将常用数据库连接代码进行提取

像常用的，经常会调用并且多个位置重复调用同样的代码，可以提取成一个方法作为一个工具使用。

工具在项目中一般存放在工具类里，方法用来完成某一功能，类中的方法完成某一主题。

工具类一般放在工具包中，包命名为util。

工具类中方法的异常一般不会捕获，而是抛出，警告使用者注意异常！

工具类中的方法可以使用静态修饰，方便使用者调用！

java
//提取前两步和关流代码到工具类中：util.JDBCUtil
public static Connection getConnection() throws Exception{
    Class.forName("oracle.jdbc.driver.OracleDriver");
    Connection conn = DriverManager.getConnection("jdbc:oracle:thin:@127.0.0.1:1521:orcl","hr","system");
    return conn;
}

public static void closeAll(Connection conn,Statement sta,ResultSet rs) throws Exception{
    if(rs!=null)rs.close();
    if(sta!=null)sta.close();
    if(conn!=null)conn.close();
}

//在DAO层就可以直接调用工具类方法进行获得连接和关流：dao.StudentDao
public List<Student> selectAllStudent(){
    PreparedStatement ps = null;
    Connection conn = null;
    ResultSet rs = null;

    List<Student> l = null;
    try {
        conn = JDBCUtil.getConnection();
        ps = conn.prepareStatement("select * from student");
        rs =ps.executeQuery();

        l = new ArrayList<Student>();
        while(rs.next()){
            l.add(new Student(rs.getInt("SID"),rs.getString("SNAME"),rs.getString("SAGE"),rs.getString("SSEX")));
        }

    } catch (Exception e) {
        e.printStackTrace();
    } finally{
        try {
            JDBCUtil.closeAll(conn, ps, rs);
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
    return l;
}

控制数据库连接次数

多次重复的与数据库建立连接将加重数据库负载并浪费连接资源

可以采用类似单例的方式将连接对象控制为一个，使程序运行过程中只会出现一个连接对象

因为只有一个连接对象所以在使用过程中不能关闭连接，方式后续使用导致出现“关闭的连接”异常

包分层 ​

dao：

持久层：与数据库交互的代码，类与表对应，一个类里处理一个表的操作为一个方法，一个方法一个SQL语句不能多了，方法不能是静态的。只与数据库交互，收到组装SQL的参数，返回数据库取到的结果。

entity/bean：

实体类：与数据库的表进行对应，还会出现与表不对应的情况（扩展类），一个类描述一个实体，类中存在get/set/toString/构造方法。用于描述一个实体。

util：

工具类：经常会调用的，重复调用的代码存放在此类，一个类代表一个主题（JDBC，编码，输入，加密），类中的方法为主题中的具体操作（JDBC：获得连接，关流，编码：编码转换，生成编码，判断编码）。

service：

服务层：服务层的一个类表示一个业务逻辑（登录，注册，转账），类中的方法全部供此逻辑使用。所有的逻辑及转换等操作全部在此层执行，是代码最多的层次。

config：

配置文件：供工具类或者程序运行所需的字符值配置存放处，一般以流的方式读取并使用Properties类解析。

main/test：

测试类：局部的区间的进行代码测试，其中存在main函数等可执行函数。只是一个临时的代码。

servlet=>collection/action：

控制层：用于于前端页面进行数据交互：收和发，和编码修改操作。仅仅用于数据收发。

作用 ​

  • 将不同功能的代码提取到不同位置，在出现异常时便于维护。
  • 并且寻找单独的功能块会更加简单。
  • 方法的增多使代码重用性增加，并有效的解耦合。

调用顺序 ​

servlet调用service

service调用dao

dao返回结果给service

service返回结果给servlet

bean传输用

util频繁调用在各处都可以用

main测试，可以切入任何点，但程序运行不会运行main中内容

书写顺序（推荐） ​

  1. 要先建好实体类，因为在后续使用中要经常用到。实体类名尽量与表名相同，字段尽量也与表相同。
  2. 然后建立DAO层的类，类名与表名相同后加Dao单词表示此类为持久层。
  3. 将DAO层的重复代码提取到Util类中。
  4. 将工具类中的配置值放到外部配置文件中，进行加载。优化连接及注册驱动的次数。注意，工具类中的异常一般是抛出的，不会捕获（提醒调用者不要出此类异常）。
  5. 使用工具类对DAO层代码进行优化。
  6. 创建一个DAO层的接口，一个接口对应一个实现类，实现类写在dao.impl包下，接口写在dao包下。
  7. 创建一个Service层，如果单一功能的操作较多，可以将一个操作作为一个类，类中的方法用来支持同一操作（做一个删除操作=>查询此用户是否存在=>查询此用户是否有子项=>删除用户）；如果单一功能的操作比较少（做一个删除操作=>发送要删除的id给dao直接删除），那就把一个模块（用户模块，订单模块）作为一个类，类中的方法来实现不同的功能。
  8. 在测试层或者Servlet层调用service接口中的内容。
