  • 字符集选为Unicode.

Oracle 11G 卸载 ​

停止Oracle服务 ​

  • 进入计算机右击>管理界面.

  • 停止所有Oracle相关的服务,右击停止.(下图表示所有的服务已经停止)

启动卸载程序 ​

  • 在开始菜单的Oracle文件夹中找到Universal Installer选项,点击卸载产品开始卸载.

  • 在弹出层中选中OraDb11g_home1之外的所有内容(子目录和兄弟目录).

  • 选中后点击删除,将会提示删除的内容.

  • 执行卸载程序.

  • 卸载完成之后,将展示本机产品清单.

  • 执行完成后关闭页面,在第一个页面点击取消.

  • 此时本机中关于Oracle的内容已经清空,开始菜单中只剩下Oracle访问网址.

注册表删除 ​

  • 进入DOS窗口,输入regedit开始设置注册表.

  • 在弹出窗口中找到HKEY_LOCAL_MACHINE\SOFTWARE文件夹,然后删除Oracle文件夹.

  • 继续在HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services中删除所有oracle开头的项.

  • 重新启动计算机,重启后删除Oracle安装目录.

MySQL 5.5.5 安装 ​

安装页面 ​

  1. 欢迎页面:

  2. 须知页面,需要勾选框后才能进行下一步:

  3. 选择安装类型,一般选择通常类型即可.

  4. 点击Install开始安装.

  5. 开始安装:

介绍页面 ​

  1. 数据库安装完成后,将自动弹出介绍页面.

  2. 勾选"开始配置MySQL",并点击Finish.

配置MySQL ​

  1. 开始配置:

  2. 在"详细配置"与"标准配置"中选择详细配置,类似自定义配置.

  3. 第一项表示开发使用,第二项表示运行使用,作为服务器,第三项表示专有服务器.他们的区别是对内存的占用不同.选择开发机器.

    Developer Machine:这是一台开发机器，许多其他应用程序都将在它上运行。mysql服务器应该只使用最少的内存。

    Server Machine:服务器应用程序将在计算机上运行。为Web/应用程序服务器选择此选项。mysql的内存使用率中等

    Dedicated MySQL Server Machine:此计算机专用于运行MySQL数据库服务器。不会运行其他服务器，如Web或邮件服务器。MySQL将使用最多所有可用内存.

  4. 第一项表示多功能数据库,第二项表示仅事务数据库,第三项表示非事务数据库.选择多功能数据库.

    Multifunctional Database:通用数据库。这将优化服务器以使用快速事务性InnoDB存储引擎和高速MyISAM存储引擎。

    Transactional Database Only:为应用服务器和事务性web应用程序优化。这将使innodb成为主要的存储引擎，注意myisam引擎仍然可以使用。

    Non-Transactional Database Only:适用于简单的Web应用程序、监视或日志记录应用程序以及分析程序。只有非事务性MyISAM存储引擎才会被激活。

  5. 选择表空间开辟的位置:

  6. 第一项表示:决策支持（dds）/olap,第二项表示联机事务处理（OLTP）,第三项表示手工装配.选择决策支出dds/olap.

    Decision Support(DDS)/OLAP:对于不需要大量并发连接的数据库应用程序，请选择此选项。将假定有20个连接。

    Online Transaction Processing(OLTP):对于高度并发的应用程序，可以选择此选项，这些应用程序一次最多有500个活动连接，例如大量加载的web服务器。

    Manual Setting:请输入并发连接的大概数量.

  7. 设置服务器端口和是否躲避防火墙检测.保持默认状态.

  8. 选择数据库语言与编码方式.

  9. 添加到系统服务和添加系统变量.

  10. 输入最高权限管理员的密码:

  11. 开始配置数据库参数(完成状态):

MySQL 5.5.5 卸载 ​

控制面板中卸载 ​

删除注册文件 ​

  • HKEY_LOCAL_MACHINE/SYSTEM/ControlSet001/services/eventlog/Application/删除MySQL
  • HKEY_LOCAL_MACHINE/SYSTEM/ControlSet002/services/eventlog/Application/删除MySQL
  • HKEY_LOCAL_MACHINE/SYSTEM/CurrentControlSet/services/eventlog/Application/删除MySQL

删除隐藏文件 ​

在C盘找到隐藏文件:Program Files和Program Files（x86）,将其中关于MySQL的内容删除.

显示隐藏文件:

  • Win7:在组织中,选择文件夹和搜索选项,显示隐藏文件.
  • Win7:点击键盘Alt,调出菜单,在工具中,找到文件夹选项,显示隐藏文件.
  • Win10:在文件夹的查看目录下可以直接勾选"显示隐藏的文件".

删除安装文件 ​

找到MySQL安装时的路径(默认存在于C:\Program Files),删除MySQL安装位置文件.

HR用户解冻 ​

  1. 通过安装程序可以直接进入浏览器中控台,或者通过开始菜单中Oracle的安装目录今入.

  2. 填入在安装时,给System分配的密码(口令),今入中控系统,开始设置管理员.

    System是数据库的最高权限管理员,可以修改和设置数据库的所有信息.安全起见不使用System用户.

    数据库中默认存在一个HR(管理员)用户,默认冻结状态.这个用户就是为了学习数据库才产生的.

    在管理的用户管理中,可以将其解锁.

  3. 点击进入HR,设置其权限和状态(设置为未冻结状态),点击变更用户按钮,完成并保存设置.
