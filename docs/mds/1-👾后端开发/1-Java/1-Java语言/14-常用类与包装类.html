常用类 ​

Object ​

  1. Object类位于java.lang包下，可以不导包直接使用（每一个java文件默认导入java.lang）。
  2. Object类是所有类的父类，所有自己手动创建或者别人写好的类都默认继承Object类。
  3. 所有的类都能使用Object类中非私有的方法。

toString() ​

  1. toString方法用于返回调用者的十六进制地址表达形式。

  2. toString方法会在**输出引用的时候自动调用！**输出的是引用，实际显示的是toString方法的返回值！

  3. 可以重写父类中的toString方法，使其不显示十六进制地址表达形式，而是显示当前对象中的内容。在输出引用的时候，可以直接看到对象中的内容。

java
public class Dog{
	String name;
	int age;
	
	public Dog(String name,int age){
		this.name = name;
		this.age = age;
	}
	//重写Object中的toString方法，用于输出对象中的内容！
	@Override
	public String toString() {
		return this.name+" is "+this.age+" years old.";
	}
}

引用可以直接与字符串相加，但是不能与数字相加。

finalize() ​

  1. finalize方法用于处理程序运行过程中产生的内存垃圾[^内存垃圾]。
  2. finalize方法在垃圾到达一定大小时自动执行，无需手动调用。

getClass() ​

  1. getClass方法用于获得程序运行时引用中的实际对象。
  2. 此方法返回一个Class类型的对象，此对象用于描述“类”的信息，并重写了toString方法显示运行时对象的类型是类还是接口，以及其包名和类名（全限定名）。

equals() ​

  1. 在Object类中，equals依然比较的是调用者与参数的地址是否相同（采用==的方式比较）。
  2. 通常情况下，会重写Object类中的equals方法，用于比较两个对象的内容是否相同。
java
//String对equals方法的的重写
public boolean equals(Object anObject) {
    //判断自反性：如果调用者与参数地址都相同，则内容一定相同
    if (this == anObject) {
        return true;
    }
    //判断参数是否是String类型或者String的子类类型，为了后期安全转换
    //如果参数都不是String家族的，则没有存储字符串的能力，不能进行比较
    if (anObject instanceof String) {
        //之所以进行转换，是为了使用其中的value属性，value属性表示字符串存储的char类型数组
        String anotherString = (String) anObject;
        int n = value.length;
        //判断调用者与参数的字符串长度是否一致，如果一致则进行下一步判断
        if (n == anotherString.value.length) {
            //遍历两个字符串所存储的char类型数组的每个元素
            char v1[] = value;
            char v2[] = anotherString.value;
            int i = 0;
            while (n-- != 0) {
                //如果有任何一个元素不同，则返回false
                if (v1[i] != v2[i])
                    return false;
                i++;
            }
            //如果循环完毕都没有返回，则在此处返回true
            return true;
        }
    }
    //如果上文中出现任何异常情况都将返回false
    return false;
}
java
//自定义equals方法
public class Dog{
	String name;
	int age;

	@Override//注解，此注解的作用是：表示此方法为重写父类的方法
	public boolean equals(Object obj) {
        //判断参数是否为空
		if(obj != null){
            //判断自反性
			if(obj == this){
				return true;
			}
            //判断是否可以强转
			if(obj.getClass() == new Dog().getClass()){
				Dog d = (Dog)obj;
                //将各个属性进行比较
				if(this.name.equals(d.name) && this.age == d.age ){
					return true;
				}
			}
		}
        //如果以上任何情况失误，则返回false
		return false;
	}
}

包装类 ​

java中存在两种数据类型：对象类型（引用类型、复合数据类型），基本类型。

包装类实际上就是对基本类型进行一次包装。

基本类型不具有属性，方法等对象类型所拥有的丰富的特殊的操作，使用包装类可以使基本类型具有对象类型的特征及使用方式。

所有的包装类，都放置在java.lang包下，可以不导包直接使用。

注意，基本类型的默认值是数值类型或者false，但是对象类型（包装类）的默认值都是null。

byteByte
shortShort
intInteger
longLong

自动拆装 ​

可以像使用基本类型一样使用包装类型。

java
//存值操作
int a = 10;
System.out.println(a);

Integer a2 = 10;
a2 = new Integer(11);
System.out.println(a2);

//取值操作
int b = a;
System.out.println(b);

Integer b2 = a2;
System.out.println(b2);

基本类型与对象类型包装类可以自由转换，转换过程称为自动拆装。

java
int a = 10;
		
Integer b = a;	//将基本类型赋值给对象类型

int c = b;		//将对象类型赋值给基本类型

System.out.println(c);

手动拆装 ​

手动拆装就是手动的将基本类型转换为对象类型，将对象类型转换为基本类型（同类型转换）。

java
//将基本类型转换为对象类型
int a = 10;
Integer i = new Integer(a);//此创建对象的过程参考a的值进行创建，将a的值赋值到对象类型中。

//将对象类型转换为基本类型
Integer i2 = new Integer(12);
int a2 = i2.intValue();//intValue方法的调用，将返回i2对象中所包含的int类型值。

System.out.println(i);

类型转换 ​

将String类型的对象中所包含的值，转换成Integer类型或者int类型。

String类型的值无法进行数学运算，转换成数值类型可以进行数学运算。并且在未来使用过程中，大量的数字都是使用String类型进行传递的，防止精度损失。

此处演示Integer的转换方式，其他包装类都有相应的方法使用。

java
String s = "123";
//如果将一个String字符串转换成数值类型，就必须保证字符串中全是数字！
		
Integer i = Integer.valueOf(s);	
//调用Integer类的valueOf方法，传入字符串，返回字符串内容中的Integer类型值。
		
int i2 = Integer.parseInt(s);	
//调用Integer类的parseInt方法，传入字符串，返回字符串内容中的int类型值。

String ​

  1. String是一个表示字符串的类，底层实际上是以char类型的数组来存储一段字符串。

  2. String有两种创建的方式：

    java
    String str1 = "hello";				//这种创建方式将字符串存放到串池中
String str2 = new String("hello");	//这种创建方式将字符串放到单独的对象中

    放到串池中的字符串：如果使用第一种方式创建字符串，系统会先在串池[^串池]中寻找是否有相同的字符串，如果有相同的字符串，则直接将此字符串的地址返回给引用，不会创建新的相同的字符串了。

    java
    String a = "hello";
String b = "hello";
System.out.println(a==b);//true

String a2 = new String("hello");
String b2 = new String("hello");
System.out.println(a2==b2);//false

  3. String类是一个final修饰的终极类，不能被任何子类继承。

常用方法 ​

返回值/方法名/参数表作用
char charAt(int index)参数表示下标，调用此方法返回指定下标中的字符。
String concat(String str)将参数中的字符串拼接到调用者后面，返回新的字符串。
boolean equals(Object anObject)重写自父类的方法，用于比较两个字符串是否内容相同。
byte[] getBytes()获得字符串的char类型数组的数字表达形式。
int indexOf(String str)返回第一次出现参数指定字符串的位置的下标。
boolean isEmpty()判断此字符串是否是一个没有字符的空字符串。
int length()返回字符串的长度，字符的个数。
String[] split(String regex)根据参数的指定，将字符串进行切割，返回一个字符串数组。
char[] toCharArray()返回字符串的char数组表达形式。
String toLowerCase()将调用者中的英文字符转换成小写。
String toUpperCase()将调用者中的英文字符转换成大写。

StringBuffer ​

效率方面：StringBuilder>StringBuffer>String

安全方面：StringBuffer>StringBuilder

与StringBuffer相似的还有一个StringBuilder类，与StringBuffer相同，但是StringBuffer线程安全，StringBuilder线程不安全！

String类创建的字符串，在进行字符串拼接的时候，会产生多个中间值。中间值没有任何引用牵引，变成垃圾。

当需要频繁的字符串拼接的时候，会产生许多垃圾。可以使用StringBuffer（存在缓冲区的String字符串）来进行字符串拼接，然后将拼接成功的字符串导出。

java
StringBuffer s = new StringBuffer("hello");		//创建一个StringBuffer对象，添加内容。
s.append("world");								//对原有字符串进行尾部追加操作。
String str = s.toString();						//将StringBuffer中的内容转换成String类型。

BigDecimal ​

作用 ​

可以使用此类解决精度损失问题，或者值过大的问题。

BigDecimal可以存储较大的值，并且在传输和使用过程中不会出现精度损失问题。

常用方法 ​

返回值/方法名/参数表方法作用
构造方法通过有参构造方法，可以初始化一个任意类型的BigDecimal对象。
BigDecimal add(BigDecimal augend)将调用者与返回值进行相加操作。
int compareTo(BigDecimal val)将调用者与参数进行比较，相等返回0，调用者大返回1，否则返回-1
BigDecimal scaleByPowerOfTen(int n)返回其数值等于 (this * 10^n) 的 BigDecimal。
double doubleValue()将此 BigDecimal 转换为 double。
